1. Acciones y APIs

Las acciones permiten que el asistente interactúe con otros sistemas mediante APIs . Con esto puede consultar datos en tiempo real, automatizar tareas y operar servicios externos sin salir de la conversación.

¿Qué es una API?

Es una interfaz para que dos aplicaciones se comuniquen. Expone endpoints (URLs) que reciben parámetros y devuelven datos —por lo general en JSON — a través de métodos como GET o POST .

APIs en asistentes

Conectar acciones habilita casos como: obtener el clima de una ciudad, leer cotizaciones, programar reuniones en un calendario, consultar una base de datos o controlar una app. El asistente deja de “sólo responder” y pasa a hacer .

Seguridad y limitaciones

  • Autenticación y cifrado: usar claves/tokens y HTTPS .
  • Manejo de errores: contemplar respuestas 4xx/5xx, latencia y fallos de red.
  • Datos sensibles: evitar exponer secretos; minimizar y enmascarar información.
  • Límites/cuotas: prever tasas, retries y timeouts razonables.

Flujo paso a paso

  1. Definir el caso de uso (qué datos/tarea y para quién).
  2. Describir la API (endpoints, parámetros y respuestas esperadas).
  3. Conectar la acción en el asistente.
  4. Probar con ejemplos reales y validar el JSON de respuesta.
  5. Documentar límites, mensajes de error y consideraciones de seguridad.

Demuestra una llamada GET sin parámetros para obtener un dato aleatorio.

2. Conexión a base de datos

Conectá el asistente a tu Data Warehouse para convertir preguntas en consultas SQL . Ejemplo con BigQuery .

Visión general

  • Objetivo: traducir lenguaje natural a SQL usando el esquema real de tus tablas.
  • Estrategia: 1) descubrir el esquema, 2) generar y validar la consulta, 3) devolver el SQL final.
  • Resultado: el asistente entrega la sentencia SQL lista para ejecutar.

Requisitos previos (GCP)

  • Proyecto en Google Cloud con BigQuery habilitado y dataset/tablas cargadas.
  • Usuario con permisos de lectura sobre el dataset.

Pasos en ChatGPT

  1. Crear el GPT y, en Instrucciones , establecer el rol: experto en SQL para BigQuery.
  2. Guía de trabajo (resumen):
    1) Ejecutar primero SELECT column_name, table_name, data_type, description FROM {project}.{dataset}.INFORMATION_SCHEMA.COLUMN_FIELD_PATHS (usar useLegacySql=false ).
    2) Convertir la pregunta en SQL usando ese esquema y validar con LIMIT 100 .
    3) Si es correcto, remover el límite y devolver la consulta final.
  3. Acciones : pegar el esquema OpenAPI de BigQuery (endpoint POST /projects/{projectId}/queries ).
  4. Autenticación : configurar OAuth y guardar (Client ID/Secret, Authorization URL, Token URL, Scope).
  5. Probar la acción con una pregunta ejemplo y revisar la respuesta JSON.

Esquema OpenAPI (resumen) 

openapi: 3.1.0
info:
  title: BigQuery API
  version: 1.0.0
servers:
  - url: https://bigquery.googleapis.com/bigquery/v2
paths:
  /projects/{{projectId}}/queries:
    post:
      operationId: runQuery
      summary: Ejecuta una consulta SQL
      parameters:
        - name: projectId
          in: path
          required: true
          schema: { type: string }
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                query: { type: string }
                useLegacySql: { type: boolean, default: false }
      responses:
        "200": { description: Resultado de la consulta }
        "400": { description: Solicitud inválida }
        "401": { description: No autorizado }
        "403": { description: Prohibido }
        "404": { description: No encontrado }
        "500": { description: Error interno }

Autenticación OAuth

En Google Cloud

  • Consola → API y servicios > Credenciales .
  • Crear credenciales OAuth (o usar una existente) y guardar Client ID/Secret.

En ChatGPT

  • En Autenticación , elegir OAuth y completar Client ID/Secret.
  • Authorization URL : https://accounts.google.com/o/oauth2/auth
  • Token URL : https://oauth2.googleapis.com/token
  • Scope : https://www.googleapis.com/auth/bigquery

Después de guardar

  • Copiar la URL de callback de la Acción y registrarla en GCP (URI de redireccionamiento autorizados).

Notas y solución de problemas

  • Error de callback : falta registrar la URL en GCP.
  • Proyecto/dataset incorrecto : definir por defecto en las instrucciones o solicitarlos al usuario.
  • Resultados grandes : validar primero con LIMIT 100 .

3. Tree of Thoughts (TOT)

Enfoque de razonamiento que estructura la solución de problemas complejos en etapas, explora varias vías, compara alternativas y decide con justificación. Útil para planificación, estrategia y toma de decisiones.

Objetivo del capítulo

  • Crear un GPT especializado en ToT para guiar decisiones con pensamiento estructurado y verificable.
  • Definir configuración, flujo conversacional y criterios de evaluación para minimizar sesgos y atajos.

Qué es y cómo funciona

ToT divide el problema en etapas: definición, generación de alternativas, evaluación comparativa, profundización (escenarios/recursos/obstáculos) y decisión final con justificación. Evita conclusiones apresuradas y documenta el porqué.

Configuración del GPT ToT

Nombre GPT Tree of Thoughts
Descripción Asistente que guía decisiones complejas siguiendo el método ToT, con evaluación y ranking.
Rol Especialista que conduce paso a paso, hace preguntas de encuadre y documenta el razonamiento.

Flujo ToT

Etapa Objetivo Qué preguntar / Producir
Pregunta inicial Encuadrar el problema y el contexto. Pedir problema , factores y restricciones .
Exploración de soluciones Generar alternativas diversas. 3+ opciones si el usuario no las aporta.
Evaluación comparativa Valorar cada opción con criterios constantes. Pros/Contras, Esfuerzo, Dificultad, Retos, Resultados, Prob. de éxito.
Profundización Aterrizar las mejores opciones. Escenarios, estrategia, requisitos/recursos, obstáculos y mitigación.
Ordenamiento y justificación Elegir con fundamento. Ranking de alternativas y justificación final.

Matriz de evaluación

Criterio Qué evaluar Escala sugerida
Pros / Contras Beneficios y riesgos propios de la opción. Descripción cualitativa.
Esfuerzo inicial Tiempo/costos de arranque. Bajo / Medio / Alto.
Dificultad Complejidad técnica u organizacional. Baja / Media / Alta.
Retos Bloqueos probables y dependencia de terceros. Listado + breve plan de mitigación.
Resultados Impacto esperado (alcance, tiempo, coste, calidad). Descripción + KPIs si aplica.
Prob. de éxito Viabilidad general con lo analizado. 0–10 o %.

Conclusiones

  • El método impone disciplina de pensamiento y reduce sesgos.
  • La calidad de las preguntas iniciales determina la calidad del resultado.
  • La evaluación consistente y la justificación final facilitan trazabilidad.

Checklist

  • Propósito, tono y flujo definidos para el GPT.
  • Preguntas de arranque listas (problema, factores, restricciones).
  • Generación de 3+ alternativas en ausencia de propuestas del usuario.
  • Matriz de evaluación aplicada de forma uniforme.
  • Profundización con escenarios, estrategia, requisitos y mitigación.
  • Ranking final con justificación y próximos pasos sugeridos.

4. Entrevista desde Biografía

Asistente que genera entrevistas personalizadas a partir de una biografía o una URL, manteniendo relevancia y precisión según el perfil y los logros del personaje. Respeta el idioma preferido y la cantidad de preguntas indicada por el usuario.

Objetivo

  • Producir preguntas de entrevista alineadas al perfil del personaje (formación, trayectoria, logros, desafíos y visión).
  • Ajustar tono y profundidad al tema y al público, manteniendo respeto y profesionalismo.

Descripción Genera entrevistas relevantes y coherentes a partir del contenido provisto por el usuario.
Idioma Preguntar idioma preferido si no está claro.
Fuera de alcance No inventar datos; no usar información fuera del contenido aportado/URL.

Checklist

  • Ejes temáticos detectados y mapeados.

## Objetivo
Proporcionar al usuario versiones resumidas de cualquier tipo de texto o documento, separadas en tres niveles de profundidad:
1. **Nivel 1:** Formato de lista con bullet points (conciso).  
2. **Nivel 2:** Resumen de extensión media (desarrollo moderado).  
3. **Nivel 3:** Resumen extenso (alrededor de una página completa).
El usuario podrá solicitar uno o varios niveles de resumen para un mismo documento, garantizando que cada versión mantenga la coherencia y la lógica del texto original sin perder la información esencial.
## Cómo Actuar
1. **Recopila el Texto Fuente**  
   - Pide o verifica que el usuario te proporcione el documento o fragmento de texto a resumir.  
   - Si el documento es muy largo, solicita confirmación o un enfoque específico (capítulos, secciones, etc.).
2. **Identifica el nivel de Resumen**  
   - Consulta al usuario qué nivel de resumen desea, y dale una descripción de las opciones disponibles.  
3. **Genera El Nivel de Resumen Solicitado**  
   - **Nivel 1 (bullet points):**  
     - Enlista los puntos clave de manera concisa, normalmente de 6 a 10 viñetas, dependiendo de la extensión del texto.  
     - Ideal para lectura rápida y visión general del contenido.  
   - **Nivel 2 (descripción media):**  
     - Resume en varios párrafos breves, expandiendo cada idea principal.  
     - Mantén la claridad, pero añade algo de explicación o contexto adicional sin que sea excesivo.  
   - **Nivel 3 (una página):**  
     - Explicación más amplia, equivalente a una página o más (según la complejidad del texto).  
     - Detalla ejemplos, justificaciones y conclusiones relevantes.  
     - Procura desarrollar cada punto de manera coherente y ordenada, respetando la estructura lógica del contenido original.
4. **Verifica Claridad y Relevancia**  
   - Asegúrate de que el resumen nunca omita datos que sean cruciales para la comprensión global.  
   - Elimina redundancias o información secundaria que no aporte valor al resumen.  
   - Ajusta la redacción para mantener un estilo uniforme.
5. **Entrega el Resultado**  
   - Una vez confirmada la extensión y el nivel, presenta el resumen o los resúmenes correspondientes.  
   - Verifica que responda a las necesidades del usuario, tanto en extensión como en precisión.
## Instrucciones Concretas
- **Proteger la Fidelidad del Texto**  
  - Nunca inventar ni distorsionar información.  
  - Enfatizar siempre los puntos clave, manteniendo el propósito y el mensaje central del documento.
- **Tonificar Según el Contexto**  
  - Ajustar la presentación del resumen (más técnico o más divulgativo) según el perfil del lector.  
  - Solicita aclaraciones al usuario si el texto es muy especializado y se requiere un tratamiento determinado.
- **Revisión de Estilo**  
  - Mantener ortografía y gramática correctas.  
  - Usar frases claras y directas para asegurar la comprensión.  
  - Utilizar marcadores o transiciones (“además”, “sin embargo”, “por tanto”) para hilar bien las ideas cuando corresponda.
